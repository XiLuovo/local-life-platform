## 1 分钟版本

你好，我主要学习和实践的方向是 Java 后端开发，技术栈以 Spring Boot、MyBatis、MySQL、Redis 为主。最近我做的核心项目是一个本地生活平台后端，它是在原有外卖项目基础上，整合了点评和秒杀系统的能力，形成了一个完整的业务闭环。

在这个项目里，我主要做了三类工作。第一类是业务整合，把原来分散的外卖、门店、优惠券和社交功能重新组织成一个统一平台。第二类是用户和业务链路改造，比如把用户登录从微信登录调整成手机号验证码登录加 JWT，这样更适合本地演示和继续扩展。第三类是技术亮点实现，比如 Redis 缓存、BitMap 签到、ZSet 点赞和 Feed 流，以及用 Lua + Redis Stream 实现高并发秒杀异步下单。

我现在比较希望进入实习岗位，把这些项目里的后端设计和实现经验继续打磨成更贴近真实业务的能力。

## 2 分钟版本

你好，我目前主要在做 Java 后端方向的学习和项目实践，技术栈主要包括 Spring Boot、MyBatis、MySQL、Redis、JWT 和 WebSocket。相比只做单点功能，我更关注怎么把一个项目做成完整业务闭环。

我最近做的代表项目是一个本地生活平台后端。这个项目最开始其实来自两个独立练手项目，一个是外卖系统，一个是点评秒杀系统。我没有简单把两个项目拼接，而是把外卖主链路保留为骨架，再把门店、优惠券、秒杀、探店笔记、关注和签到这些能力整合进去，做成一个完整的平台。

在这个过程中，我主要负责几件事。第一是业务整合与模块设计，明确哪些能力是主链路，哪些是亮点模块。第二是用户体系改造，把原来的微信登录改成手机号验证码登录加 JWT 鉴权，降低项目演示门槛。第三是 Redis 能力落地，包括门店缓存、BitMap 连续签到、Set 关注关系、ZSet 点赞和 Feed 流。第四是高并发秒杀方案，我把秒杀券下单从同步处理升级成了 Lua + Redis Stream 的异步方案，先在 Redis 层做库存和一人一单校验，再通过 Stream 异步落库和做失败补偿。

我觉得这个项目让我成长最多的一点，是学会了怎么从“功能能跑”走到“业务能讲、技术能讲、结构也更完整”。所以我现在找后端实习时，希望把这种从业务到技术的整合理解继续用到更真实的项目中。
